### 1. The Mobile Device Token Deduplicator

**The Problem:** Your React Native app collects push notification tokens from different active sessions. Due to network retries, the payload contains duplicate, outdated, or corrupted values (`null` or `undefined`).

Write a function `cleanTokens(sessions)` that takes an array of session objects, extracts all valid string tokens, removes duplicates, and returns them sorted alphabetically.

#### The JavaScript / TypeScript Solution:

```typescript
interface Session {
  sessionId: string;
  token?: string | null;
  meta?: { isActive: boolean };
}

export function cleanTokens(sessions: Session[]): string[] {
  if (!sessions || !Array.isArray(sessions)) return [];

  const uniqueTokens = new Set<string>();

  for (const session of sessions) {
    // Safe check for null, undefined, empty strings, and session state
    if (
      session &&
      typeof session.token === "string" &&
      session.token.trim().length > 0
    ) {
      if (session.meta?.isActive !== false) {
        uniqueTokens.add(session.token.trim());
      }
    }
  }

  // Convert Set back to array and sort alphabetically
  return Array.from(uniqueTokens).sort();
}
```

- **Why TechCorp asks this:** Mobile apps frequently deal with cleaning up dirty payload data before storing it locally or sending it to registration servers.

### 2. State Diff Tracker (Object Comparison)

**The Problem:** To prevent unnecessary re-rendering of a heavy React Native component list, you need to track exactly which values changed between the `previousState` object and the `currentState` object.

Write a function `getChangedKeys(previousState, currentState)` that returns an array of keys whose values are different. Assume a flat object structure (key-value pairs of strings or numbers).

#### The JavaScript / TypeScript Solution:

```javascript
function getChangedKeys(previousState, currentState) {
  if (!previousState || !currentState) return [];

  const changedKeys = [];

  // Combine all unique keys from both states to ensure we catch additions/deletions
  const allKeys = new Set([
    ...Object.keys(previousState),
    ...Object.keys(currentState),
  ]);

  for (const key of allKeys) {
    if (previousState[key] !== currentState[key]) {
      changedKeys.push(key);
    }
  }

  return changedKeys.sort();
}
```

- **Why TechCorp asks this:** Emulates deep lifecycle tuning and virtualized list optimizations inside complex React Native ecosystems.

## Coding (Standard) 2: Asynchronous API Processing & Aggregation

_This section requires querying a live paginated endpoint (usually `https://jsonmock.hackerrank.com`) using `async/await`. You must parse multiple pages and aggregate numerical data._

### 3. Paginated API Filtering (The "Classic" HackerRank)

**The Problem:** Fetch transaction details from the paginated endpoint: `https://jsonmock.hackerrank.com/api/transactions?page=X`.

Calculate the total transaction amount spent across **all pages** by a specific user (`userId`) where the transaction status is `"success"`.

#### The JavaScript / Node.js Solution:

```javascript
// Note: HackerRank provides 'node-fetch' natively in its execution context
const fetch = require("node-fetch");

async function getTotalUserSpent(targetUserId) {
  let currentPage = 1;
  let totalPages = 1;
  let totalSpent = 0;

  try {
    while (currentPage <= totalPages) {
      const url = `https://jsonmock.hackerrank.com/api/transactions?page=${currentPage}`;
      const response = await fetch(url);

      if (!response.ok) break; // Gracefully exit if an API page errors out

      const jsonPayload = await response.json();

      // Dynamically lock down total pages from the initial payload metadata
      if (currentPage === 1) {
        totalPages = jsonPayload.total_pages || 1;
      }

      const transactions = jsonPayload.data || [];

      for (const item of transactions) {
        // Ensure type safety since IDs can arrive as strings or numbers from mock databases
        if (
          String(item.userId) === String(targetUserId) &&
          item.status === "success"
        ) {
          // Extract numerical value, stripping out currency formatting symbols like '$' or ',' if present
          const sanitizedAmount =
            typeof item.amount === "string"
              ? parseFloat(item.amount.replace(/[^0-9.]/g, ""))
              : item.amount;

          totalSpent += sanitizedAmount || 0;
        }
      }

      currentPage++;
    }
  } catch (error) {
    // Return 0 instead of crashing the assessment execution path completely
    return 0;
  }

  return Math.round(totalSpent);
}
```

- **Why TechCorp asks this:** This is the most standardized real-world setup HackerRank uses. It assesses how cleanly you handle Promises, API boundaries, and unexpected string-to-number mutations.

### 4. API Request Race with Timeout Fallback

**The Problem:** Mobile devices face unstable internet connectivity. Write an async helper function `fetchWithTimeout(url, timeoutMs)` that races a standard HTTP `fetch` call against a timeout promise. If the endpoint doesn't respond within `timeoutMs`, the function should reject with a custom message `"REQUEST_TIMEOUT"`.

#### The JavaScript Solution:

```javascript
async function fetchWithTimeout(url, timeoutMs = 5000) {
  // 1. Create the timeout anchor
  const timeoutTrigger = new Promise((_, reject) => {
    setTimeout(() => reject(new Error("REQUEST_TIMEOUT")), timeoutMs);
  });

  // 2. Race the network fetch execution block against the timer anchor
  try {
    const networkResponse = await Promise.race([fetch(url), timeoutTrigger]);

    if (!networkResponse.ok) throw new Error("FETCH_ERROR");
    return await networkResponse.json();
  } catch (error) {
    // Bubble up error to be intercepted by parent component boundary wrappers
    throw error;
  }
}
```
