## 🏗️ खंड 1: ऑब्जेक्ट-ओरिएंटेड प्रोग्रामिंग (ओओपी) और वैचारिक प्रश्नोत्तर
*⏱️ 6 मिनट पढ़ें*

### 1. सार वर्ग तात्कालिकता
एक अमूर्त वर्ग को सीधे चालू नहीं किया जा सकता है। यह आंशिक कार्यान्वयन और हस्ताक्षर वाले बेस क्लास के रूप में कार्य करता है। आपको इसे इनहेरिट करना होगा और इसके सभी अमूर्त तरीकों को एक उपवर्ग में लागू करना होगा।

```java
abstract class Animal {
 abstract void sound();
}
// Animal a = new Animal(); // ❌ Error: cannot be instantiated directly

class Dog extends Animal {
 void sound() {
 System.out.println("Bark");
 }
}
Animal dog = new Dog(); // ✅ Works (Polymorphism)
```

### 2. इंटरफ़ेस बनाम सार वर्ग

| सार वर्ग | इंटरफ़ेस |
| :--- | :--- |
| पूरी तरह से कार्यान्वित तरीके और राज्य हो सकते हैं। | मुख्य रूप से एक अनुबंध के रूप में कार्य करता है (जावा 8+ डिफ़ॉल्ट तरीकों की अनुमति देता है)। |
| कंस्ट्रक्टर्स का समर्थन करता है। | कंस्ट्रक्टर्स का समर्थन नहीं करता. |
| एकल वंशानुक्रम (वर्ग केवल एक वर्ग का विस्तार कर सकता है)। | एकाधिक वंशानुक्रम (वर्ग एकाधिक इंटरफ़ेस लागू कर सकता है)। |
| सामान्य व्यवहार या कोड संरचना साझा करने के लिए उपयोग किया जाता है। | असंबंधित वर्गों में सामान्य क्षमताओं को परिभाषित करने के लिए उपयोग किया जाता है। |

### 3. बहुरूपता
बहुरूपता विधियों को उन्हें कॉल करने वाले वर्ग उदाहरण के आधार पर अलग-अलग व्यवहार करने की अनुमति देती है।

```javascript
class Animal {
 sound() { console.log("Generic sound"); }
}
class Dog extends Animal {
 sound() { console.log("Bark"); }
}
```

## 📦 धारा 2: मोबाइल डेटा संरचनाएं और व्यावहारिक परिदृश्य
*⏱️ 8 मिनट पढ़ें*

- ** टर्न-आधारित क्रियाओं (राउंड रॉबिन) के लिए Queue (परिपत्र Queue )**: टर्न-आधारित गेम या राउंड-रॉबिन सीपीयू शेड्यूलिंग के लिए आदर्श। सक्रिय खिलाड़ी को कतार से हटाएं, उनकी बारी निष्पादित करें, और $O(1)$ समय में उन्हें वापस पूंछ पर पंक्तिबद्ध करें।
- **अनुक्रमिक भंडारण के लिए सरणी**: निरंतर मेमोरी स्थान $O(1)$ अनुक्रमण की अनुमति देते हैं।
- **इतिहास और पूर्ववत संचालन के लिए स्टैक**: एक लास्ट-इन-फर्स्ट-आउट ( LIFO ) stack पृष्ठ navigation बैक बटन (सक्रिय दृश्य को पॉप करना) और Ctrl+Z पूर्ववत बफ़र्स का प्रबंधन करता है।
- ** शेड्यूलिंग के लिए Queue **: फर्स्ट-इन-फर्स्ट-आउट ( FIFO ) queue इनकमिंग कॉल शेड्यूलिंग या एपीआई अनुरोध निष्पादन कतारों को निष्पक्ष रूप से व्यवस्थित करता है।
- **फ़ोन बुक लुकअप के लिए हैश मैप**: संपर्कों से मैप किए गए फ़ोन नंबर तत्काल $O(1)$ कुंजी खोज की अनुमति देते हैं।
- **फ़ाइल सिस्टम के लिए ट्री**: फ़ोल्डर्स और सबफ़ोल्डर्स के साथ फ़ाइल निर्देशिकाओं को एन-एरी ट्री संरचनाओं का उपयोग करके तैयार किया गया है।
- ** BFS में Queue / DFS में स्टैक **: BFS FIFO Queue का उपयोग करके स्तर-दर-स्तर नोड्स को ट्रैवर्स करता है। DFS LIFO स्टैक (या रिकर्सन stack ) का उपयोग करके गहरे नीचे के पथों को पार करता है।
- **रिएक्ट रिकंसिलिएशन में ट्री**: रिएक्ट रिकंसिलिएशन वैचारिक रूप से एक ट्री पदानुक्रम का उपयोग करके वर्चुअल DOM को मैप करता है और रैखिक $O(N)$ समय में परिवर्तनों का पता लगाता है।

## 🛠️ धारा 3: परिदृश्य-आधारित कोडिंग एल्गोरिदम (40+ समस्याएं)
*⏱️ 12 मिनट पढ़ें*

### ऐरे एल्गोरिदम
- **पहला गैर-दोहराया जाने वाला तत्व**: आवृत्ति मानचित्र को पॉप्युलेट करने के लिए एक बार लूप करें, फिर 1 की गिनती के साथ पहला तत्व ढूंढने के लिए दूसरी बार लूप करें।
- **दूसरी सबसे बड़ी संख्या**: दो चर ( `first` और `second` ) को ट्रैक करते हुए सरणी को पार करें। यदि `num > first` है, तो `first` को `second` पर शिफ्ट करें और `first` अपडेट करें।
- **शून्य को अंत तक ले जाएं**: एक राइट पॉइंटर `insertPos` का उपयोग करें। सरणी के माध्यम से पुनरावृत्त करें; यदि कोई तत्व गैर-शून्य है, तो उसे `insertPos` पर तत्व के साथ बदलें और `insertPos` बढ़ाएँ।
- **ऐरे को K द्वारा घुमाएँ**: संपूर्ण ऐरे को उलट दें, पहले $K$ तत्वों को उलट दें, और फिर शेष $N-K$ तत्वों को $O(1)$ सहायक स्थान में उलट दें।
- **गुम संख्या ढूंढें**: $\frac{N(N+1)}{2}$ का उपयोग करके $1$ से $N$ तक अपेक्षित अंकगणितीय योग की गणना करें और वास्तविक सरणी योग घटाएं।

### स्ट्रिंग एल्गोरिदम
- **पैलिंड्रोम की जांच करें**: चरित्र समानता पर जोर देते हुए, बाहरी छोर से two pointers चलाएं ( `left` , `right` ) केंद्र में मिलते हुए।
- **उल्टे शब्द**: स्ट्रिंग को रिक्त स्थान से विभाजित करें, बीच में two pointers मीटिंग का उपयोग करके तत्वों को स्वैप करें, और उन्हें वापस जोड़ें।
- **डुप्लिकेट वर्ण**: मानचित्र के अंदर वर्णों की घटनाओं की गणना करें और 1 से अधिक गिनती वाली कुंजियों को फ़िल्टर करें।
- **अनाग्राम जांचें**: पहली स्ट्रिंग से आवृत्ति गणना मानचित्र बनाएं। दूसरी स्ट्रिंग में प्रत्येक वर्ण के लिए कमी गिना जाता है। यदि कोई गिनती नकारात्मक तक पहुंचती है या वर्ण गायब है, तो वे विपर्यय नहीं हैं।

### HashMap एल्गोरिदम
- **उच्चतम आवृत्ति तत्व**: एक आवृत्ति मानचित्र बनाएं और अधिकतम गिनती रखने वाली कुंजी को ट्रैक करें।
- **समूह अनाग्राम**: प्रत्येक स्ट्रिंग को वर्णानुक्रम में क्रमबद्ध करें। किसी सरणी में मिलान करने वाले विपर्यय को समूहित करने के लिए क्रमबद्ध स्ट्रिंग को हैश मैप कुंजी के रूप में उपयोग करें।

### स्टैक और Queue एल्गोरिदम
- **मान्य कोष्ठक**: वर्णों के माध्यम से पुनरावृत्त करें। ओपनिंग ब्रैकेट को stack पर पुश करें। कोष्ठकों को बंद करने के लिए, stack से पॉप करें और दावा करें कि वे मेल खाते हैं।
- **न्यूनतम स्टैक डिज़ाइन**: एक सहायक stack ( `minStack` ) रखें जो न्यूनतम मान संग्रहीत करता है। `push(val)` पर, यदि मान `minStack` के शीर्ष से कम या उसके बराबर है, तो इसे `minStack` पर धकेलें।
- **स्टैक्स का उपयोग करके Queue लागू करें**: दो स्टैक का उपयोग करें: एन्क्यू के लिए `stack1` और डीक्यू के लिए `stack2` । यदि `stack2` खाली है, तो `stack1` से सभी तत्वों को पॉप करें और उन्हें `stack2` पर पुश करें।

### लिंक्ड सूची और ट्री एल्गोरिदम
- **रिवर्स लिंक्ड लिस्ट**: नोड लिंक को स्थान पर स्वैप करने के लिए तीन पॉइंटर्स ( `prev` , `curr` , `nextNode` ) का उपयोग करें।
- **लूप का पता लगाएं (फ्लोयड का चक्र)**: धीमे पॉइंटर को 1 नोड से और तेज़ पॉइंटर को 2 नोड से घुमाएँ। यदि वे मिलते हैं, तो एक चक्र अस्तित्व में है।
- ** BST सत्यापन**: पुनरावर्ती रूप से सत्यापित करें कि नोड्स गतिशील `min` और `max` रेंज के बीच आते हैं।

### 2. वास्तुशिल्प डिजाइन पैटर्न
उच्च परीक्षण योग्यता के साथ एक बड़े React Native कोडबेस (कोड की 100k+ लाइनें) को बनाए रखने के लिए, आपको संरचित आर्किटेक्चर पैटर्न लागू करना चाहिए:

- ** Monorepos ( Yarn कार्यस्थान/टर्बो)**:
- अलग-अलग पैकेजों में चिंताओं को अलग करें: `@app/shared` (प्रकार, उपयोगिताएँ), `@app/ui` (पुन: प्रयोज्य परमाणु डिजाइन घटक), और `@app/core` ( state management , एपीआई क्लाइंट)।
 - iOS , Android और वेब प्लेटफॉर्म के बीच कोड शेयरिंग को अधिकतम करता है।
- ** Clean Architecture और फ़ीचर-प्रथम निर्देशिका संरचना**:
 - फ़ाइलों को तकनीकी फ़ाइल प्रकार ( `components` , `redux` , `hooks` ) के बजाय डोमेन सुविधा (जैसे, `features/authentication` , `features/checkout` ) के अनुसार व्यवस्थित करें।
 - **स्वच्छ परतें**:
 - **डोमेन परत (इकाइयाँ और नियम)**: शुद्ध व्यावसायिक तर्क, UI frameworks से स्वतंत्र।
 - **डेटा परत (रिपॉजिटरी और स्रोत)**: स्थानीय भंडारण और एपीआई एकीकरण को संभालता है।
 - **प्रस्तुति परत (घटक और हुक)**: UI लेआउट प्रस्तुत करने पर सख्ती से ध्यान केंद्रित करता है।

### 3. एज-केस सिचुएशन हैंडलिंग

#### ए. कम मेमोरी/रैम सीमाएं
मोबाइल डिवाइस (विशेष रूप से पुराने Android फोन) जल्दी से आउट-ऑफ-मेमोरी (OOM) crashes को ट्रिगर कर सकते हैं:
- ** Hermes कचरा संग्रहण**: Hermes एक जेनरेशनल मार्क-एंड-स्वीप जीसी का उपयोग करता है। स्पाइक्स को रोकने के लिए, उच्च-आवृत्ति निष्पादन लूप के अंदर बड़े सरणी या ऑब्जेक्ट आवंटित करने से बचें।
- ** FlashList के माध्यम से सेल रीसाइक्लिंग **: `FlatList` को `@shopify/flash-list` से बदलें। FlashList सेल दृश्यों को अनमाउंट करने और उन्हें पुन: बनाने के बजाय, मूल आवंटन को कम करके उनका पुनर्चक्रण करता है।
- **इमेज कैश एविक्शन**: इमेज कैश को मेमोरी में कैप करें और ऐप के बैकग्राउंड में प्रवेश करने पर उन्हें गतिशील रूप से साफ़ करें:

```javascript
 import FastImage from 'react-native-fast-image';
 // Clear memory cache on low-memory warnings or background transitions
 FastImage.clearMemoryCache();
 ```

#### बी. कम नेटवर्क/ऑफ़लाइन स्थिति
सेल्युलर नेटवर्क बार-बार बंद हो जाते हैं। ऐप्स को नेटवर्क परिवर्तनों को शालीनता से संभालना होगा:
- **बैकऑफ़ के साथ क्वेरी पुनः प्रयास**: घातांकीय बैकऑफ़ के साथ क्वेरी पुनः प्रयास करने के लिए एपीआई कॉल रैपर कॉन्फ़िगर करें:

```javascript
 const { data } = useQuery({
 queryKey: ['profile'],
 queryFn: fetchProfile,
 retry: 3,
 retryDelay: (attempt) => Math.min(1000 * 2 ** attempt, 30000)
 });
 ```

- **परसिस्टेंट आउटबॉक्स Queue **: लंबित ऑफ़लाइन म्यूटेशन को स्थानीय स्टोरेज ( MMKV या SQLite ) में स्टोर करें और डिवाइस के वापस ऑनलाइन होने पर उन्हें क्रमिक रूप से प्रोसेस करें।
- **संघर्ष समाधान (वेक्टर घड़ियाँ)**: यदि कोई उपयोगकर्ता किसी ऑर्डर को ऑफ़लाइन संशोधित करता है, तो पेलोड में एक वेक्टर घड़ी टाइमस्टैम्प `{ clientVersion: 5, serverVersion: 4 }` जोड़ें। यदि आधार संस्करण डेटाबेस से मेल खाता है तो सर्वर मान्य करता है; यदि कोई विरोध होता है, तो यह एक मर्ज समाधान रणनीति लागू करता है (उदाहरण के लिए, "सर्वर संस्करण रखें" या "आइटम मर्ज करें")।

#### सी. बड़ी संपत्ति और स्टार्टअप प्रदर्शन
बड़ा बंडल आकार ऐप स्टार्टअप को धीमा कर देता है:
- **एसेट स्लाइसिंग**: Xcode एसेट कैटलॉग ( `.xcassets` ) और Android ड्रॉएबल फ़ोल्डर्स का उपयोग करें। app store केवल लक्ष्य डिवाइस से मेल खाने वाली छवि घनत्व ( `@2x` , `@3x` ) परोसता है।
- **आलसी लोडिंग घटक**: द्वितीयक स्क्रीन (जैसे, सेटिंग्स, प्रोफ़ाइल उप-पृष्ठ) को लोड करने को स्थगित करने के लिए `React.lazy()` या गतिशील आयात का उपयोग करें जब तक कि उपयोगकर्ता उन पर नेविगेट न कर ले।
- **छवियों को वेबपी में कनवर्ट करें**: बंडल फ़ुटप्रिंट को कम करने के लिए कच्ची पीएनजी/जेपीजी संपत्तियों को संपीड़ित वेबपी छवियों से बदलें।

#### डी. पृष्ठभूमि कार्य
बैटरी ख़त्म किए बिना पृष्ठभूमि में कार्य चलाएँ:
- **हेडलेसजेएस ( Android )**: एक जावा पृष्ठभूमि सेवा शुरू करें जो ऐप बंद होने पर भी जावास्क्रिप्ट कार्य चलाती है।
- **बैकग्राउंड फ़ेच ( iOS )**: स्थानीय डेटाबेस को समय-समय पर अपडेट करने के लिए iOS सिस्टम द्वारा निर्धारित मूल बैकग्राउंड फ़ेच कार्यों का उपयोग करें।

### 3. Node.js और Express.js मोबाइल बैकएंड एपीआई
मोबाइल अनुरोधों को प्रॉक्सी करने वाले एपीआई गेटवे को प्रमुख सुरक्षा और दर-सीमित सुविधाओं को लागू करना चाहिए:

```javascript
const express = require('express');
const jwt = require('jsonwebtoken');
const helmet = require('helmet');
const rateLimit = require('express-rate-limit');

const app = express();

// Secure Headers
app.use(helmet());

// Rate Limiter
const apiLimiter = rateLimit({
 windowMs: 15 * 60 * 1000, // 15 mins
 max: 100 // max 100 requests per IP per window
});
app.use('/api/', apiLimiter);

// Mobile Authentication Middleware
function authenticateJWT(req, res, next) {
 const authHeader = req.headers['authorization'];
 const token = authHeader && authHeader.split(' ')[1];

 if (!token) return res.sendStatus(401);

 jwt.verify(token, process.env.ACCESS_TOKEN_SECRET, (err, user) => {
 if (err) return res.sendStatus(403);
 req.user = user;
 next();
 });
}
```

### 1. पिरामिड मैपिंग का परीक्षण

- **यूनिट परीक्षण (मज़ाक)**:
 - पृथक कार्यों, उपयोगिता सहायकों और custom hooks के परीक्षण पर ध्यान केंद्रित करता है।

```javascript
 import { renderHook, act } from '@testing-library/react-hooks';

 test('should increment counter', () => {
 const { result } = renderHook(() => useCounter());

 result.current.increment();

 });
 ```

- **एकीकरण परीक्षण ( React Native परीक्षण पुस्तकालय - आरएनटीएल)**:

```javascript
 import { render, fireEvent, screen } from '@testing-library/react-native';

 test('submits username and password', () => {
 const mockSubmit = jest.fn();
 ```

### 1. डायग्नोस्टिक प्रोफाइलर
जब उपयोगकर्ता सुस्त रेंडरिंग या crashes की रिपोर्ट करते हैं, तो कारण का निदान करने के लिए इन tools का उपयोग करें:
- **फ़्लिपर (प्रदर्शन लॉग और लेआउट)**:
 - रिएक्ट घटक ट्री को प्रोफाइल करें और राज्य अपडेट का निरीक्षण करें।
 - नेटवर्क ट्रैफ़िक का विश्लेषण करें और डेटाबेस क्वेरीज़ को ट्रैक करें।
- ** Xcode Instruments ( iOS प्रोफाइलिंग)**:
 - **आवंटन**: समय के साथ रैम मेमोरी फ़ुटप्रिंट की निगरानी करें।
 - **लीक**: उन ऑब्जेक्ट आवंटन का पता लगाएं जो आवश्यकता न होने के बाद भी मेमोरी में बने रहते हैं।
- ** Android Profiler ( Android Studio )**:
 - सीपीयू उपयोग को ट्रैक करें और थ्रेड शेड्यूलिंग का निरीक्षण करें।
 - सीढ़ी जैसे रैम पैटर्न की पहचान करने के लिए मेमोरी उपयोग की निगरानी करें ( memory leaks इंगित करता है)।

### 2. मेमोरी लीक का पता लगाना और उसे ठीक करना
 React Native में मेमोरी leaks आमतौर पर जावास्क्रिप्ट और नेटिव मेमोरी के बीच bridge परत में होती है:
- **लक्षण**: जैसे-जैसे आप स्क्रीन के बीच आगे-पीछे नेविगेट करते हैं, मेमोरी का उपयोग लगातार बढ़ता जाता है, कभी भी इसकी आधार रेखा पर वापस नहीं आता है।
- **सामान्य कारण: अस्वच्छ घटना श्रोता**:

```javascript
 // ❌ LEAK: AppState listener remains in memory after component unmounts
 useEffect(() => {
 AppState.addEventListener('change', handleAppStateChange);
 }, []);

 // ✅ FIX: Clean up the event listener
 useEffect(() => {
 const subscription = AppState.addEventListener('change', handleAppStateChange);
 return () => subscription.remove();
 }, []);
 ```

- **सामान्य कारण: UI दृश्य नोड्स बनाए रखा (फ्लिपर/लीक)**:
यदि कोई मूल दृश्य JS में अनमाउंट किया गया है लेकिन मूल सूचक द्वारा संदर्भित रहता है, तो यह leak मेमोरी होगी। रिटेन graph का पता लगाने और लटकते संदर्भ का पता लगाने के लिए Xcode Instruments का उपयोग करें।

### 3. State Management पुस्तकालय तुलना

- ** Redux Toolkit (आरटीके)**:
 - **प्रकार**: ग्लोबल सेंट्रलाइज्ड स्टोर।
 - **तंत्र**: actions और reducers का उपयोग करके अनुमानित स्थिति अपडेट।
 - **इसके लिए सर्वश्रेष्ठ**: बड़े पैमाने पर क्लाइंट-साइड UI कॉन्फ़िगरेशन, उपयोगकर्ता सेटिंग्स और जटिल फॉर्म।
- ** MobX (अवलोकन योग्य स्थिति)**:
 - **प्रकार**: प्रतिक्रियाशील/परिवर्तनशील अवस्था।
 - **तंत्र**: अवलोकन और प्रतिक्रियाओं का उपयोग करता है। जब वेरिएबल परिवर्तन पर निर्भर होते हैं तो UI घटकों को स्वचालित रूप से अपडेट करता है।
 - **इसके लिए सर्वश्रेष्ठ**: जटिल नेस्टेड रिश्तों के लिए उच्च-आवृत्ति अपडेट की आवश्यकता होती है (उदाहरण के लिए रीयल-टाइम मैपिंग)।
- ** Recoil / जोताई**:
 - **प्रकार**: परमाणु अवस्था।
 - **तंत्र**: अवस्था (परमाणुओं) के छोटे, पृथक टुकड़ों का उपयोग करता है जिन्हें स्वतंत्र रूप से जोड़ा और अद्यतन किया जा सकता है।
 - **इसके लिए सर्वश्रेष्ठ**: बड़े ऐप्स को संपूर्ण घटक ट्री को पुन: प्रस्तुत किए बिना बारीक अपडेट की आवश्यकता होती है।

## 📦 धारा 8: परिनियोजन पाइपलाइन और स्टोर रिलीज़
*⏱️ 8 मिनट पढ़ें*

### 1. iOS App Store रिलीज़ प्रबंधन
- ** iOS लक्ष्य उन्नयन**: Set न्यूनतम iOS लक्ष्य (जैसे iOS 15.0) दोनों Podfile और Xcode बिल्ड सेटिंग्स के अंदर।
- **प्रमाणपत्र एवं प्रावधान**:
 - **विकास प्रमाणपत्र**: स्थानीय परीक्षण के लिए ऐप्स पर हस्ताक्षर करने के लिए उपयोग किया जाता है।
 - **वितरण प्रमाणपत्र**: TestFlight और App Store सबमिशन के लिए ऐप्स पर हस्ताक्षर करने के लिए उपयोग किया जाता है।
 - **प्रावधान प्रोफ़ाइल**: रन को अधिकृत करने के लिए आपके developer प्रमाणपत्र और ऐप आईडी को लिंक करता है।
- ** Fastlane स्वचालन**:
 अपना ऐप बनाने और स्वचालित रूप से TestFlight पर अपलोड करने के लिए Fastlane का उपयोग करें:

```bash
 fastlane gym --scheme "ProductionApp" --export_method "app-store"
 fastlane pilot upload_to_testflight
 ```

- **गोपनीयता घोषणापत्र ( `PrivacyInfo.xcprivacy` )**:
 App Store समीक्षा पास करने के लिए सभी SDKs ( Sentry , फायरबेस) और एपीआई की घोषणा करें जो Xcode के Privacy Manifest के अंदर उपयोगकर्ता डेटा को ट्रैक करते हैं।

### 2. Android Play Store रिलीज प्रबंधन
- **संकलन एवं लक्ष्य SDK **:
 Set लक्ष्य SDK संस्करण `android/app/build.gradle` के अंदर:

```groovy
 android {
 compileSdkVersion 34
 defaultConfig {
 targetSdkVersion 34
 }
 }
 ```

- **हस्ताक्षर कुंजी ( Keystore )**:
 `gradle.properties` में कॉन्फ़िगर की गई सुरक्षित कीस्टोर फ़ाइल का उपयोग करके अपने उत्पादन बिल्ड पर हस्ताक्षर करें:

```properties
 MYAPP_RELEASE_STORE_FILE=my-release-key.keystore
 MYAPP_RELEASE_KEY_ALIAS=my-key-alias
 ```

- ** Google Play रिलीज़ ट्रैक**:
 - **आंतरिक साझाकरण**: Google समीक्षा के बिना परीक्षकों के साथ तुरंत साझाकरण होता है।
 - **बंद/खुला परीक्षण**: बीटा परीक्षकों के लिए रिलीज़ बिल्ड। थोड़े समीक्षा समय की आवश्यकता है.
 - **प्रोडक्शन ट्रैक**: ऐप को आम जनता के लिए जारी करें। सभी उपयोगकर्ताओं तक पहुंचने से पहले Sentry पर crash दरों की निगरानी के लिए प्रगतिशील रोलआउट (उदाहरण के लिए 5% ➡️ 20% ➡️ 100%) का उपयोग करें।
